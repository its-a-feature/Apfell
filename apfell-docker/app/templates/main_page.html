Welcome to Apfell

Check out the documentation or GitHub Repos for more information on the process of creating Apfell, code repos, and blog posts on updates.

Remember: You must set your current operation in "Manage Operations" -> "Operations Management" before you can use most of the features in Apfell.

What is this Apfell framework?

  • Apfell is meant to be an extendable user interface that allows operators to create/adjust/control agents in an easier fashion than starting from scratch.
  • For information on agent specific commands, check out: Help -> Command Help
  • For information on how to create a new agent or leverage some internal features, check out: hooking features

How can I quickly start using this framework as an operator?

  • Manage Operations -> Operations Management
  • Select an operation to "Make Current". You'll see this reflected in the top navigation bar
  • Manage Operations -> C2 Profiles Management
  • Potentially update the configuration for the profile via "Edit Files", download, modify, and re-upload
  • Click "Start"
  • Create Components -> Create Base Payload
  • Select your C2 Profile
  • Fill in any required parameters (some may be auto-populated with hints or default values)
  • Select a payload type that supports this profile
  • Supply a name for your payload
  • Supply a tag for your payload. This tag will auto-populate for the description of any callbacks based on this payload.
  • Select any/all commands you want to be initially available in this payload. Depending on the payload type, you can load more commands after you get a callback.
  • Click "Submit". This will either submit the tasking to the corresponding container or, if it's an external payload type, then it just registers the payload and data in the database.
  • Manage Operations -> Payload Management
    • Click "Download" to download the payload to your system
    • Click "Config" to view the metadata about your payload
    • Click "Host" to host the payload with any name
  • There are many ways to do this, and they are very specific to the payload type. When creating the payload, you'll be presented with the "execution help" provided by the author of the payload type
  • At any point, when you click the "config" for a payload on the "Payload Management" page, you'll be able to see the execution help there as well
  • Click "Interact" and start issuing commands
  • Be sure to check out the following:
    • Split Callbacks
    • Click the green + sign to add comments
    • Add scripting capabilities to automatically process your output via the "Browser Script Management" page
    • Search (via all output or all command parameters)
